Checked Build ASSERTs

This topic contains a list of common ASSERTs encountered by driver writers.

See How the Checked Build Indicates a Problem for some tips on how to handle these asserts (and others not listed).

The routine listed in the Routine Called column is the most common routinethat driver writers or system components would call to provoke this error. Some of the routines listed below are documented routines that drivers call. Others are internal routines that only system components can call. Keep in mind that calling some other function from your driver might result in the called function internally calling one of the listed functions, which could in turn issue the ASSERT.
